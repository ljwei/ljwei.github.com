Java虚拟机一--简介

发表于 2016-03-23   |   分类于 Java   |  

编译与解释

目前,流行的编程语言有很多,其中Java和C++是最常拿来相互比较的.Java和C++有很多相似的地方,比如面向对象,静态类型等等.但是两者有一个本质的区别,就是Java是解释型的,而C++是编译型的.由于这一区别,可以解释很多Java和C++的不同之处,比如:Java的可移植性好,Java的执行效率低(以前,Java效率确实很低,但是现在JVM经过不断优化,与C++几乎没有差别,甚至更快)等等.

编译:
利用编译器从源语言编写的源程序产生目标程序的过程。这里的目标程序一般是机器指令,编译的结果与CPU平台有关,因此要在不同的平台中运行,就需要重新编译.

解释:
与编译一次就产生所有的目标代码不同,解释器是按行读取源程序,然后解释执行.

目前的很多解释型语言,例如Java和Python,也会有编译的过程,只不过是先由编译器(对Java来说,就是javac.exe)生成与平台无关的中间代码,然后由Java虚拟机来解释执行,这样可以做到与平台无关,即一次编写,到处运行.

JDK,JRE,JVM

这是java中经常涉及到的一些概念,要清楚了解这些概念,就要了解Java的整个体系.Sun官方定义的Java技术体系包括以下几个部分:

  • Java语言本事
  • 各种硬件平台上的Java虚拟机
  • Class文件格式
  • Java API类库
  • 来自商业机构和开源社区的第三方Java类库

其中,把Java程序设计语言,Java虚拟机,和Java API类库这三部分统称为JDK(Java Development Kit),JDK是用于支持Java开发的最小环境;把Java API类库中的Java SE API子集和Java虚拟机这两部分统称为JRE(Java Runtime Environment),JRE是支持Java程序运行的标准环境;JVM是Java Virtual Machine（Java虚拟机）的缩写，JVM是一种用于计算设备的规范，它是一个虚构出来的计算机，是通过在实际的计算机上仿真模拟各种计算机功能来实现的。

在Java的发展历史中,出现过很多的JVM,Sun只是定义Java虚拟机规范,很多公司根据这个规范,都有自己实现的JVM,这些JVM各有特点.其中最常见的有:Sun HotSpot VM,BEA JRockit,IBM J9 VM.后来Sun和BEA都被Oracle收购,目前在已发布的JDK中使用的就是HotSpot.
